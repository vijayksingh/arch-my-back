# Architecture Design Tool

A collaborative architecture design and documentation tool built with React, Vite, Convex, and React Flow. Design system architectures with an interactive canvas while maintaining synchronized documentation.

## Features

- **Interactive Canvas**: Drag-and-drop architecture components, shapes, and connections using React Flow
- **Structured Documentation**: Organize requirements, schema definitions, API specs, and low-level designs
- **Real-time Sync**: All changes are instantly persisted to Convex database
- **Authentication**: Secure sign-in with Google OAuth or email/password
- **Multi-workspace**: Manage multiple architecture projects simultaneously
- **Sections & Linking**: Group canvas elements into sections and link them to documentation blocks

## Tech Stack

- **Frontend**: React 19, TypeScript, Vite
- **Canvas**: React Flow (XYFlow)
- **Backend**: Convex (real-time database and backend)
- **Authentication**: @convex-dev/auth with Google OAuth and Password providers
- **Styling**: Tailwind CSS 4, Radix UI components
- **State Management**: Zustand

## Prerequisites

- **Node.js**: v18 or higher
- **pnpm**: v9 or higher (install with `npm install -g pnpm`)
- **Convex Account**: Free account at [convex.dev](https://convex.dev)
- **Google OAuth Credentials** (optional): For Google sign-in

## Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd arch-app
   ```

2. **Install dependencies**
   ```bash
   pnpm install
   ```

3. **Set up Convex**

   Create a Convex account and initialize your project:
   ```bash
   npx convex dev
   ```

   This command will:
   - Create a new Convex project (or link to existing)
   - Generate a deployment URL
   - Start the Convex development server
   - Create the `.env.local` file with `VITE_CONVEX_URL`

4. **Configure environment variables**

   Your `.env.local` file should contain:
   ```bash
   # Convex deployment URL (auto-generated by `npx convex dev`)
   VITE_CONVEX_URL=https://your-deployment.convex.cloud
   ```

5. **Set up Google OAuth** (Optional - for Google sign-in)

   a. Go to [Google Cloud Console](https://console.cloud.google.com/)

   b. Create a new project or select an existing one

   c. Enable the Google+ API

   d. Navigate to **Credentials** → **Create Credentials** → **OAuth 2.0 Client ID**

   e. Configure the OAuth consent screen if prompted

   f. Select **Web application** as the application type

   g. Add authorized redirect URIs:
      - Development: `http://localhost:5173/api/auth/callback/google`
      - Production: `https://your-domain.com/api/auth/callback/google`

   h. Copy the **Client ID** and **Client Secret**

   i. Add them to the [Convex Dashboard](https://dashboard.convex.dev):
      - Navigate to your project → **Settings** → **Environment Variables**
      - Add: `AUTH_GOOGLE_ID` = your-client-id
      - Add: `AUTH_GOOGLE_SECRET` = your-client-secret

   **Note**: Environment variables for OAuth are set in the Convex Dashboard, NOT in `.env.local`.

## Data Model

### Tables

- **users**: User account information
- **workspaces**: User workspaces with view settings and preferences
- **designs**: Canvas data (nodes, edges, sections)
- **blocks**: Documentation blocks (text, requirements, schema, API, LLD)
- **auth tables**: Managed by @convex-dev/auth (sessions, accounts, etc.)

See `convex/schema.ts` for the complete schema definition.

## Architecture Overview

### Authentication Flow
1. User signs in via Google OAuth or email/password
2. @convex-dev/auth handles authentication and session management
3. Convex auth tables store user sessions and account linkage
4. User ID is stored in the `users` table for application data

### Canvas + Document Sync
1. Canvas nodes and edges are stored in the `designs` table
2. Documentation blocks are stored in the `blocks` table
3. Sections link canvas regions to documentation blocks via `sectionId`
4. All updates are real-time via Convex subscriptions

### State Management
- **Zustand stores**: Client-side state (theme, UI preferences)
- **Convex queries**: Server state with real-time updates
- **React Flow**: Canvas state managed by XYFlow

## Deployment

### Deploy to Vercel (or similar)

1. **Build the frontend**
   ```bash
   pnpm build
   ```

2. **Deploy Convex functions**
   ```bash
   npx convex deploy --prod
   ```

   This creates a production Convex deployment and outputs the production URL.

3. **Set production environment variables**

   In your hosting provider (e.g., Vercel):
   ```bash
   VITE_CONVEX_URL=https://your-production-deployment.convex.cloud
   ```

4. **Update Google OAuth redirect URIs**

   Add your production domain to the authorized redirect URIs in Google Cloud Console:
   ```
   https://your-domain.com/api/auth/callback/google
   ```

5. **Deploy your frontend** to your hosting provider

### Convex Production Environment Variables

Set these in the Convex Dashboard for your production deployment:
- `AUTH_GOOGLE_ID` - Google OAuth Client ID
- `AUTH_GOOGLE_SECRET` - Google OAuth Client Secret

## Key Features Explained

### Workspaces
Each user can have multiple workspaces. Workspaces store:
- View mode preference (canvas-only, document-only, or both)
- Active canvas tool (cursor, select, shapes, text)
- Document editor mode (edit or preview)

### Canvas Sections
Sections group related canvas nodes together:
- Visual bounding box on the canvas
- Linked to documentation blocks via `linkedBlockId`
- Helps organize large architecture diagrams

### Documentation Blocks
Five types of blocks:
- **Text**: Markdown-based free-form documentation
- **Requirements**: Structured requirement items
- **Schema**: Database schema definitions
- **API**: API endpoint specifications
- **LLD**: Low-level design documents

## Troubleshooting

### Convex connection issues
- Ensure `npx convex dev` is running
- Verify `VITE_CONVEX_URL` in `.env.local` matches your deployment URL
- Check the Convex Dashboard for deployment status

### Google OAuth not working
- Verify redirect URIs in Google Cloud Console match your app URL exactly
- Ensure `AUTH_GOOGLE_ID` and `AUTH_GOOGLE_SECRET` are set in Convex Dashboard
- Check browser console for CORS or redirect errors

### Development server conflicts
- Ensure no other processes are using ports 5173 (Vite) or 3000 (Convex dev dashboard)
- Try clearing browser cache and localStorage
- Restart both dev servers
